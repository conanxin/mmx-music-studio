# MiniMax 音乐创作台 — 产品设计文档

## 1. 产品目标

打造一个**简约、有设计感、中文界面**的音乐创作工具。用户通过填入自己的 MiniMax Token Plan Key，即可使用 MiniMax 的音乐生成能力，生成音乐、在线试听、下载 MP3、管理历史作品。

核心价值：
- **BYOK（Bring Your Own Key）** — 用户自持 Key，自控数据
- **自托管** — 完全开源，可部署在任何环境
- **简约创作体验** — 无冗余功能，专注音乐生成核心流程

## 2. 用户画像

| 用户类型 | 场景 | 需求 |
|---------|------|------|
| 独立音乐人 | 快速生成 BGM / 灵感片段 | 速度快、操作简单、中文界面 |
| 内容创作者 | 为视频/播客配乐 | 下载方便、格式标准 |
| 开发者 | 集成音乐生成能力 | 开源自托管、API 可扩展 |
| 音乐爱好者 | 探索 AI 音乐 | 体验门槛低、界面美观 |

## 5. 四种生成模式详解

### 5.1 纯音乐（BGM）

**输入：**
- 音乐描述（文字）：如「深夜编程、专注、轻松爵士」
- 风格标签：安静/明亮/梦幻/电影感/电子/钢琴/吉他/Lo-fi
- 高级设置：模型版本、输出格式、音质

**输出：**
- MP3 文件
- 波形封面图

### 5.2 自动成歌

**输入：**
- 歌曲主题（文字）：如「关于夏天的分手故事」
- 风格标签：流行/民谣/摇滚/R&B/电子/古风
- 高级设置：模型版本

**输出：**
- 自动生成的歌词（MiniMax 生成）
- 完整歌曲 MP3

### 5.3 歌词成歌

**输入：**
- 歌词（用户提供）
- 风格标签：同自动成歌
- 高级设置：模型版本

**输出：**
- 完整歌曲 MP3（用户词 + MiniMax 曲）

### 5.4 参考改编

**输入：**
- 参考音频文件（MP3/WAV/FLAC）
- 改编描述（文字）：如「把这首歌改成更抒情的版本」
- 风格标签

**输出：**
- 改编后的 MP3

## 6. 状态定义

| 状态码 | 状态名 | 说明 |
|--------|--------|------|
| `idle` | 等待中 | 未发起生成 |
| `generating` | 生成中 | 任务进行中 |
| `completed` | 已完成 | 可试听/下载 |
| `failed` | 失败 | 生成失败 |
| `key_missing` | Key 未设置 | 用户未填 Key |
| `key_connected` | Key 已连接 | Key 验证通过 |
| `quota_unknown` | 额度未知 | 未查询额度 |
